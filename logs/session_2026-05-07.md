## 2026-05-07 - 训练 loss 计算说明

### Prompt

用户询问训练时的 loss 是如何计算的。

### Response

回答说明：当前有 epoch loss 的只有 LSTM dQdV 预测器，RF、retention bridge、direct interval baseline 是 sklearn 树模型，没有逐 epoch loss。LSTM 训练 loss 是标准化后的 dQdV 多输出 MSE：先基于训练窗口的 dQdV 目标 `y_train_flat` 对每个输出维度计算 mean/std，其中输出维度为 `horizon * 5`；每个 batch 中将 `y_batch` 标准化为 `(y_batch - target_mean) / target_std`，模型输出也处于该标准化空间；使用 `nn.MSELoss()` 计算 `pred` 与 `scaled_y` 的均方误差。batch loss 是 batch 内所有样本和所有输出维度的平均 MSE；epoch 的 `train_loss`/`valid_loss` 是按 batch size 加权后的平均。早停和 best checkpoint 依据 `valid_loss` 是否比历史 best 至少降低 `min_delta`，达到 `patience` 后停止。该 loss 不是 retention loss，也不是原始尺度 dQdV MSE；自然尺度 dQdV MSE/R2 和 retention MSE/R2 是训练后评估指标。

## 2026-05-07 - dQdV 多特征 loss 与标准化口径

### Prompt

用户询问：dQdV 有多个特征时最终 loss 如何计算，采用什么标准化方法，是否基于 train 数据做标准化。

### Response

回答说明：最终 LSTM loss 是对标准化后所有输出维度的普通平均 MSE，未给不同 dQdV 特征或不同 horizon step 设置额外权重。输出向量长度为 `horizon * 5`；`horizon=1` 时是 compact5 五个特征，`horizon=K` 时是 K 个未来步的 compact5 展平向量。标准化方法是 z-score，且只基于训练窗口目标 `y_train_flat` 拟合：`mean = y_train.mean(axis=0)`，`std = y_train.std(axis=0)`，若某维 `std < 1e-6` 则置为 1.0。训练与验证 loss 都用同一组 train mean/std 将目标转换为 `(y_batch - target_mean) / target_std`，模型输出也在该标准化空间；`nn.MSELoss()` 默认 reduction 为 mean，因此 batch loss 会平均所有样本与所有输出维度。epoch loss 是按 batch size 加权的 batch loss 平均。预测阶段再用 `pred_scaled * target_std + target_mean` 还原到原始 dQdV 尺度。

## 2026-05-07 - 第一阶段 dQdV 后如何预测 retention

### Prompt

用户询问：第一阶段利用工况预测 dQdV 后，是如何预测 retention 的。

### Response

回答说明：第二阶段使用一个独立的 retention bridge 模型。该 bridge 不是神经网络，而是 `SimpleImputer(strategy="median") + RandomForestRegressor` 的 sklearn Pipeline。训练时在训练集 cycle 级真实数据上拟合，输入为真实 dQdV compact5 加上下文列 `initial_c_rate`、`switch_soc_percent`、`post_switch_c_rate`、`cycles`、`cycle_index_norm`，标签为 `retention`。第一阶段 LSTM/RF 预测出未来 dQdV 后，脚本按每个窗口和每个 horizon step 构造 bridge 输入表：把预测 dQdV compact5 填入目标列，同时带入目标 cycle 对应的 policy/cycle context、`q_ref` 和真实 retention 用于评估。然后调用 bridge model 得到 `pred_retention`，并额外计算 `pred_q_discharge = pred_retention * q_ref`。评估时按 model/set/horizon 对 `pred_retention` 与 `retention_true` 计算 MSE/RMSE/MAE/R2。需要强调：当前 bridge 训练用真实 dQdV，推理/验证时可喂预测 dQdV；并且 bridge 输入不是 dQdV-only，而是 dQdV + policy/cycle context。
